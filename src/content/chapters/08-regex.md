## 정규표현식이란?

정규표현식(Regular Expression, regex)은 **텍스트 패턴을 표현하는 미니 언어**입니다. "이런 형태의 텍스트를 찾아줘"라고 파이썬에게 요청하는 방법입니다.

> 비유하면: 정규표현식은 **만능 검색어**입니다. "숫자 2자리 + 콜론 + 숫자 2자리"처럼 구체적인 패턴을 검색할 수 있습니다.

우리 프로젝트에서는 ASS 태그 제거, 카라오케 타이밍 추출, LRC 타임스탬프 검증에 정규표현식을 사용합니다.

## `re` 모듈 기초

```python
import re
```

### `re.compile()` — 패턴 미리 컴파일

자주 사용하는 패턴은 미리 컴파일해두면 성능이 좋아집니다:

```python
# ass_parser.py — 모듈 레벨에서 한 번만 컴파일
TAG_PATTERN = re.compile(r'\{[^}]*\}')
K_TAG_PATTERN = re.compile(r'\\[Kk][fo]?(\d+)')
```

```python
# validate_lrc.py
LRC_HEADER = re.compile(r'^\[(font|title|artist)\]')
LRC_TIMESTAMP = re.compile(r'^\[(\d{2}):(\d{2})\.(\d{2})\]')
```

## 패턴 문법

### 기본 문자

| 패턴 | 의미 | 예시 매치 |
|------|------|-----------|
| `\d` | 숫자 한 개 | `0`, `9` |
| `\w` | 문자/숫자/밑줄 | `a`, `3`, `_` |
| `\s` | 공백 문자 | ` `, `\t`, `\n` |
| `.` | 아무 문자 (줄바꿈 제외) | `a`, `1`, `!` |

### 수량자

| 패턴 | 의미 | 예시 |
|------|------|------|
| `*` | 0개 이상 | `\d*` → `""`, `"123"` |
| `+` | 1개 이상 | `\d+` → `"1"`, `"123"` |
| `?` | 0개 또는 1개 | `\d?` → `""`, `"1"` |
| `{n}` | 정확히 n개 | `\d{2}` → `"00"`, `"15"` |
| `{n,m}` | n개 이상 m개 이하 | `\d{1,3}` → `"1"`, `"123"` |

### 위치와 그룹

| 패턴 | 의미 |
|------|------|
| `^` | 줄의 시작 |
| `$` | 줄의 끝 |
| `(...)` | 그룹 (캡처) |
| `[...]` | 문자 클래스 |
| `[^...]` | 부정 문자 클래스 |
| `\|` | 또는 |

## 우리 프로젝트의 정규표현식 분석

### 1. ASS 태그 제거 패턴

```python
TAG_PATTERN = re.compile(r'\{[^}]*\}')
```

이 패턴을 분해하면:
- `\{` — 여는 중괄호 `{` (리터럴)
- `[^}]*` — `}` 가 **아닌** 문자 0개 이상
- `\}` — 닫는 중괄호 `}` (리터럴)

매치 예시:
- `{\K432}` → 매치
- `{\K12}` → 매치
- `{\title}` → 매치

```python
TAG_PATTERN.sub('', r'{\K432}여행을 {\K88}떠나요')
# 결과: '여행을 떠나요'
```

### 2. 카라오케 K 태그 추출 패턴

```python
K_TAG_PATTERN = re.compile(r'\\[Kk][fo]?(\d+)')
```

분해:
- `\\` — 역슬래시 `\` (리터럴)
- `[Kk]` — `K` 또는 `k`
- `[fo]?` — `f` 또는 `o` (0개 또는 1개)
- `(\d+)` — 숫자 1개 이상 (**캡처 그룹**)

매치 예시:
- `\K432` → 매치, 그룹1 = `"432"`
- `\k12` → 매치, 그룹1 = `"12"`
- `\Kf100` → 매치, 그룹1 = `"100"`

### 3. LRC 타임스탬프 검증 패턴

```python
LRC_TIMESTAMP = re.compile(r'^\[(\d{2}):(\d{2})\.(\d{2})\]')
```

분해:
- `^` — 줄의 시작
- `\[` — 여는 대괄호 `[`
- `(\d{2})` — 숫자 2자리 (분) — **캡처 그룹 1**
- `:` — 콜론
- `(\d{2})` — 숫자 2자리 (초) — **캡처 그룹 2**
- `\.` — 마침표
- `(\d{2})` — 숫자 2자리 (센티초) — **캡처 그룹 3**
- `\]` — 닫는 대괄호 `]`

매치 예시:
- `[00:15.96]여행을` → 매치, 그룹 = (`"00"`, `"15"`, `"96"`)

### 4. LRC 헤더 검증 패턴

```python
LRC_HEADER = re.compile(r'^\[(font|title|artist)\]')
```

- `^` — 줄의 시작
- `\[` — `[`
- `(font|title|artist)` — "font" 또는 "title" 또는 "artist"
- `\]` — `]`

## `re` 모듈의 주요 메서드

### `.sub()` — 치환

```python
TAG_PATTERN.sub('', '{\K432}여행을 떠나요')
# '여행을 떠나요' — 매치된 부분을 빈 문자열로 치환
```

### `.search()` — 검색

문자열 어디서든 패턴을 찾습니다. 찾으면 Match 객체, 못 찾으면 `None` 반환.

```python
m = LRC_TIMESTAMP.search('[00:15.96]여행을 떠나요')
if m:
    minutes = m.group(1)  # "00"
    seconds = m.group(2)  # "15"
    centis  = m.group(3)  # "96"
```

### `.match()` — 시작부터 매칭

문자열의 **시작**부터 매치하는지 확인합니다.

```python
m = LRC_HEADER.match('[title:여행의 노래]')
if m:
    print(f"헤더 발견: {m.group(1)}")  # "title"
```

### `.finditer()` — 모든 매치 찾기

```python
text = r'{\K432}여행을 {\K88}떠나요 {\K120}~'
for m in K_TAG_PATTERN.finditer(text):
    print(f"위치: {m.start()}-{m.end()}, 값: {m.group(1)}")
# 위치: 1-6, 값: 432
# 위치: 12-16, 값: 88
# 위치: 22-27, 값: 120
```

우리 프로젝트에서 카라오케 프리롤을 추출할 때:

```python
# ass_parser.py
def extract_karaoke_preroll(text):
    """첫 번째 K 태그의 값(프리롤)을 센티초로 반환"""
    tags = list(TAG_PATTERN.finditer(text))
    if not tags:
        return 0
    first_tag = tags[0]
    k_match = K_TAG_PATTERN.search(first_tag.group())
    if k_match:
        return int(k_match.group(1))
    return 0
```

## Match 객체의 메서드

| 메서드 | 설명 | 예시 |
|--------|------|------|
| `.group()` | 전체 매치 | `m.group()` → `"[00:15.96]"` |
| `.group(1)` | 첫 번째 캡처 그룹 | `m.group(1)` → `"00"` |
| `.start()` | 매치 시작 위치 | `m.start()` → `0` |
| `.end()` | 매치 끝 위치 | `m.end()` → `10` |

## 원시 문자열 `r"..."` 이 필요한 이유

정규표현식에는 역슬래시(`\`)가 많이 사용됩니다. 파이썬 문자열에서도 `\`는 특수 문자(예: `\n`=줄바꿈)의 시작이므로, 충돌이 발생합니다.

```python
# ❌ 문제: \d가 파이썬에서 해석될 수 있음
pattern = re.compile('\d+')

# ✅ 원시 문자열: 역슬래시를 그대로 유지
pattern = re.compile(r'\d+')
```

`r'...'`을 사용하면 `\d`가 파이썬이 아닌 **정규표현식 엔진**에서 해석됩니다.

## 핵심 정리

| 메서드 | 용도 | 반환 |
|--------|------|------|
| `re.compile(r'패턴')` | 패턴 컴파일 | Pattern 객체 |
| `.sub('대체', 문자열)` | 치환 | 새 문자열 |
| `.search(문자열)` | 검색 | Match 또는 None |
| `.match(문자열)` | 시작 매치 | Match 또는 None |
| `.finditer(문자열)` | 모든 매치 | 반복 가능한 Match들 |
| `.group(n)` | 캡처 그룹 | 문자열 |
